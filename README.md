# 🏠 Домашняя кухня CRM v3

Мобильная CRM для учёта заказов домашней еды.

**Возможности:**
- 📅 Календарь заказов с цветными статусами и временем
- 👥 Клиенты с историей и долгами
- 🍽️ Меню с категориями и наценкой
- 📦 Склад: закупки + списания + остатки
- 💰 Финансы: балансы по счетам (наличные/карта/кредит мужа)
- 💵💳 Оплаты с выбором способа (нал/карта)
- 🤝 Учёт кредита мужа с историей погашений
- 🎯 Скидки %/₸, частичная оплата, перенос заказов

## 🚀 Часть 1. Supabase (база данных)

### 1.1 Создать проект

1. Откройте [supabase.com](https://supabase.com) → **Sign in** (через GitHub удобнее всего)
2. **New Project**
3. Заполните:
   - **Name**: `kitchen-crm`
   - **Database password**: сгенерируйте надёжный пароль и **сохраните его**
   - **Region**: выберите ближайший (для Катара — `West EU` или `South Asia`)
   - **Plan**: Free
4. Нажмите **Create new project** — подождите ~2 минуты пока проект развернётся

### 1.2 Создать таблицы

1. В левом меню: **SQL Editor**
2. **New query**
3. Откройте файл `supabase/schema.sql` из этого проекта, скопируйте всё содержимое
4. Вставьте в SQL Editor → **Run** (или `Ctrl+Enter`)
5. Должно появиться: **Success. No rows returned**

### 1.3 Получить ключи

1. Слева внизу: **Settings** (шестерёнка) → **API**
2. Скопируйте и сохраните:
   - **Project URL** (выглядит как `https://abcdefgh.supabase.co`)
   - **anon public** ключ (длинная строка, начинается с `eyJ...`)

⚠️ **НЕ используйте `service_role` ключ** — он секретный.

## 🐙 Часть 2. GitHub (хостинг)

### 2.1 Создать репозиторий

1. Откройте [github.com](https://github.com) → справа сверху **+** → **New repository**
2. Заполните:
   - **Repository name**: `kitchen-crm` (запомните это имя!)
   - **Public** (для бесплатного GitHub Pages)
   - **НЕ** добавляйте README, .gitignore — будут конфликты
3. **Create repository**

### 2.2 Загрузить файлы

**Вариант А: через GitHub Web UI (проще)**

1. На странице репозитория нажмите **uploading an existing file**
2. Перетащите ВСЕ файлы и папки из `kitchen-crm-v3/`
3. **Important:** убедитесь что скрытые папки `.github/` и `.gitignore` тоже загружены
4. **Commit changes**

**Вариант Б: через командную строку**

```bash
cd kitchen-crm-v3
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/kitchen-crm.git
git push -u origin main
```

### 2.3 Указать имя репозитория в коде

⚠️ Если назвали репозиторий **НЕ** `kitchen-crm`:

Откройте `vite.config.js` и поменяйте:
```js
base: "/kitchen-crm/",  // → замените на имя вашего репо
```

### 2.4 Добавить Supabase ключи в Secrets

1. На странице репо: **Settings** → **Secrets and variables** → **Actions**
2. **New repository secret** — добавьте два:
   - Name: `VITE_SUPABASE_URL` — Value: ваш Project URL
   - Name: `VITE_SUPABASE_ANON_KEY` — Value: ваш anon ключ

### 2.5 Включить GitHub Pages

1. **Settings** → **Pages**
2. **Source**: выберите **GitHub Actions**
3. Сохранять ничего не надо — настройка применяется автоматически

### 2.6 Запустить деплой

1. **Actions** в верхнем меню репозитория
2. Если workflow не запустился сам — сделайте любой коммит (например измените README)
3. Подождите ~2 минуты пока **build → deploy** выполнятся (зелёные галочки)
4. Адрес сайта будет:
   ```
   https://ВАШ_ЛОГИН.github.io/kitchen-crm/
   ```

Если хотите редактировать код локально:

```bash

# Клонировать репозиторий
```

## 🐛 Решение проблем

**"Не заданы Supabase-ключи"**
→ Проверьте что в **GitHub → Settings → Secrets** добавлены оба секрета с правильными именами `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`. После добавления **перезапустите Actions** (Actions → последний run → Re-run).

**Белая страница после деплоя**
→ Проверьте что в `vite.config.js` `base` совпадает с именем вашего репозитория (со слешами вокруг). Например для `kitchen-crm` это `"/kitchen-crm/"`.

**Не могу добавить заказ — ничего не сохраняется**
→ Откройте консоль браузера (F12) и посмотрите ошибку. Скорее всего не выполнен SQL-скрипт или не дано разрешение на запись (RLS-политики в схеме настроены, но проверьте что schema.sql выполнился без ошибок).

**Хочу изменить дизайн / цвета**
→ Откройте `src/index.css` — все цвета вверху в `:root { ... }`. Замените `--accent: #d4622a` на любой другой.

**Хочу добавить новые статусы / категории / валюты**
→ Статусы и метки в `src/helpers.js` (объекты `STATUS` и `SOURCE_LABEL`). Валюта — в функции `cur()` там же.

## 🔒 Безопасность

Этот шаблон рассчитан на **личное использование** (один пользователь):
- В `schema.sql` стоят политики "allow_all" — любой с anon ключом может читать/писать
- Это безопасно пока вы один знаете URL сайта и не публикуете anon-ключ в открытом виде
- anon ключ всё равно попадает в собранный билд — это нормально, он публичный

**Если нужна авторизация** (несколько пользователей):
1. В Supabase → Authentication → включите Email/Password или OAuth
2. В schema.sql замените `using (true)` на `using (auth.uid() = user_id)` и добавьте колонку `user_id` в таблицы
3. В коде добавьте `supabase.auth.signIn(...)` экран

Это уже отдельная тема — спросите когда понадобится.

## 🆙 Обновление приложения

Чтобы обновить версию:
1. Замените файлы в репозитории на новые
2. Сделайте коммит → GitHub Actions автоматически пересоберёт и задеплоит
3. **Данные в Supabase сохраняются** — обновление кода их не трогает

Если в новой версии меняется схема БД — будет отдельный SQL миграционный файл.
